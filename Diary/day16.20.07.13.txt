今天将视频传送到华为的IoTEdge上，由于我的gocv是通过挂载docker来操作的,即 docker run --rm -it -v [mygocode]:/app [dockerimage] sh -c 'cd /app;go build;/app/app'来操作的，其中--rm表示运行完就销毁容器，-it表示进入容器内部控制台( This example runs a container named test using the debian:latest image. The -it instructs Docker to allocate a pseudo-TTY connected to the container’s stdin; creating an interactive bash shell in the container. In the example, the bash shell is quit by entering exit 13. This exit code is passed on to the caller of docker run, and is recorded in the test container’s metadata.) 指示docker分配一个与容器的stdin即标准输入相连的伪tty即终端，从而在容器中创建交互式的bash，而-v表示--volum（Bind mount a volume）表示绑定安装一系列的卷即物理磁盘。所以-v可以表示将我的本地路径挂载到容器内部的路径，当容器内部没有该目录时，会在容器内部创建目录。	在镜像前的代表OPTIONS，即运行的方法，而镜像后的为COMMAND即命令创建出来的容器具体要干什么，sh -c 表示进入容器内部的控制台，之后的'xx;xx'则是在容器TTY运行的bash指令。
